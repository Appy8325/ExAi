## Replaced KPI plumbing

The public exhibitor dashboard had a live endpoint but it could not execute against the current schema. Its aggregation now uses `exhibitor_relationship_notes`, `lead_submissions.attendee_user_id`, `lead_submissions.submitted_at`, `latest_interaction_at`, and current lead-intelligence fields. This restores the existing live KPI path without new endpoint or duplicate query.

## Remaining documented limitations

- The platform Admin dashboard is intentionally a static operations shell. `AdminModule` has no controller, repository, telemetry source, or platform-admin metrics contract, so organization/user counts, uptime, service state, alerts, and operational events cannot be truthfully calculated from the application schema. No values were fabricated and no speculative observability endpoint was added.
- The attendee profile schema stores company, title, and industry. Demo seed metadata now also contains each company’s official website and networking description; exposing those fields in attendee profile screens requires a product-approved schema/API addition.
